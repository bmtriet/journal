Hướng đi tốt hơn HTML thuần: Astro + Content Collections

01/03/2026 · Vision · 5 phút đọc

Nếu mục tiêu là: chỉ viết bài rồi push, thì HTML thuần sẽ nhanh chạm trần vì mỗi bài đều phải sửa tay template, menu và danh mục.

Hướng mình đề xuất là chuyển sang Astro + Content Collections. Lúc đó bạn chỉ thêm file nội dung (Markdown hoặc JSON), còn layout, nav, category page và danh sách bài mới sẽ được build tự động.

Tạo post bằng JSON có ổn không?

Ổn, nếu bạn muốn đồng bộ dữ liệu để tái sử dụng cho nhiều chỗ (web, app, newsletter). Nhưng với blog cá nhân, mình khuyên:

  • Markdown + frontmatter cho phần thân bài (viết nhanh, đọc dễ).
  • JSON cho metadata mở rộng (tags, series, featured, related).

Cách hybrid này vừa dễ viết, vừa đủ linh hoạt để mở rộng sau này.

Vì sao Astro hợp với nhu cầu của bạn

  • Build ra static HTML nên chạy ngon trên GitHub Pages.
  • Dùng Tailwind rất mượt, giữ được style minimal hiện tại.
  • Content Collections có schema, tránh lỗi quên field khi đăng bài.
  • Tự động sinh trang danh mục và RSS/SEO dễ hơn HTML thuần.

Flow sau khi migrate

  1. Tạo file mới trong `src/content/posts/` (md hoặc json).
  2. Commit + push.
  3. GitHub Actions build & deploy, không sửa template/nav thủ công.

Kết luận: để viết lâu dài và scale nội dung, Astro là bước nâng cấp hợp lý nhất cho blog này.
